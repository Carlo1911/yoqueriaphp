1.0 Release June 20, 2015

Updating

OneUI will receive many free updates in the future adding many new awesome features, new fresh pages as well as third-party plugin upgrades. One of the the main goals was to make sure the update progress is going to be as easy as possible. Due to its nature (template), updating can become an unpleasant task.

This is why I've included a folder (01 - OneUI Get Started) in the package which I encourage you to use in your project. It contains all vital framework files minimized and ready to use in prodution. This way you will be able to update OneUI just by replacing the assets/ folder with the new one. If there is an important markup change that you will also have to make in your HTML, don't worry since I will mention it in the changelog.

Feel free to make any overwrites in other files or even export your own assets/ folder using Grunt tasks. Just make sure that any additions will be made in your own (eg LessCSS) files, so that is easier for you to upgrade frameworks files when an update is made available.

In general, the following core folders/files will have to be updated with each update:

  • fonts/*
  • css/
    • themes/*
    • bootstrap.min.css
    • oneui.min.css
  • js/
    • plugins/*
    • js/oneui.min.js

If you are using the PHP, HTML or AngularJS version (they all use the original full files) then the following core folders/files will have to be updated:

  • fonts/*
  • less/*
  • css/
    • themes/*
    • bootstrap.min.css
    • oneui.css
  • js/
    • core/*
    • plugins/*
    • js/app.js
    • js/directives.js (found only in AngularJS version and needs to be updated as well)

Moreover, if you are using Grunt, remember to update the following files in your projects root folder (notice that AngularJS version uses different files from the other 2 versions):

  • Gruntfile.js
  • package.json
When a new update becomes available, you will be able to download it from http://themeforest.net/downloads. You can check the checkbox under the product ( Get notified by email if this item is updated) or follow me on Twitter to stay up to date about the availability of new updates.
Remember that its always a good idea to backup your work before updating to a new version!

Bootstrap

Bootstrap is one of the most popular and full featured front-end frameworks. It is easy to use and it offers a large variety of CSS as well as JavaScript components. OneUI uses this framework at its core and builds on top its unique layout and features. It completely redesigns the original components and extends them to a large degree, featuring a modern and clean design.

If you are unfamiliar with the framework, its official website will help you out a lot but if you just want to start quickly, you can easily replicate any feature or component I already present in OneUI pages. If you find a feature in the official Bootstrap Docs you would like to use, feel free to, since they are all available in OneUI.

Grunt Tasks

Grunt is a JavaScript Task Runner. It enables you to automate repetitive tasks like minification, compilation, unit testing, linting, etc, which makes your life easier. In OneUI, I have included a Gruntfile which can help you live compile LessCSS files as you work and also enables you to export your own custom, minimized and production ready OneUI framework. To use it with the source code (HTML or PHP version), you will first have to:

  1. Download and install Node.js

  2. Install grunt-cli globally by running the command npm install -g grunt-cli in your systems console

  3. Install all required Grunt plugins in your projects folder by running the command npm install in your systems console - You will have to run it from your projects root folder which will include src/ folder, Gruntfile.js and package.json files.

After you successfully completed the previous 3 steps, you will be able to run the following commands in your systems console from your projects root folder:

Command Description
grunt watch

It will constantly watch for any changes made to the LessCSS files (src/assets/less/*) and will auto compile to CSS (src/assets/css/*) every time a change has been made.
grunt build

It will prepare and build your custom, minimized and production ready OneUI framework. After it completes, you will be able to find it under a new dist/ folder in your projects root folder. The same command is used for building the assets/ folder found in 01 - OneUI Get Started.
grunt build (AngularJS version)

In this version, it will prepare and build your custom, minimized and production ready AngularJS web application (not only framework's core files). After it completes, you will be able to find it under a new dist/ folder in your projects root folder.

LessCSS

OneUI was built from scratch with LessCSS. It is a powerful CSS pre-processor which extends the CSS language, adding features that allow variables, mixins, functions and many other techniques, enabling you to make CSS that is more maintainable, themable and extendable.

If you would like to extend the framework and add your own overwrites or UI components, feel free to use the LessCSS files included in the source (HTML and PHP version) under src/assets/less/* folder. You can also use the included Grunt tasks to auto compile to CSS every time you make a change. It is highly recommended that you create your own files for your changes, so that is easier for you to update the original files when an update gets released.

JavaScript

OneUI uses jQuery, Bootstrap JS framework as well as some other vital third-party jQuery plugins at its core. It also supports many more third-party plugins which you can include per page if you need them. Their style will match templates design.

Lets have a better look at the JavaScript file structure:

File Description
app.js (AngularJS version)

The main file containing AngularJS module creation, router/general configuration, custom functions and all vital controllers for our web application.
directives.js (AngularJS version)

All our custom directives can be found here. They offer custom functionality and helpers for initializing various jQuery plugins.
controllers.js (AngularJS version)

All our custom controllers for our pages can be found here. They include generic examples for showcasing a way of using more complex jQuery plugins which require custom code/data to initialize.
app.js

The main file containing all custom JS code needed for OneUI features including layout and blocks API. It also includes various helper functions which you can call per page for enabling extra features (eg Table Tools) or repetitive init code of various third party plugins.
app.js

The main file containing all custom JS code needed for OneUI features including layout and blocks API. It also includes various helper functions which you can call per page for enabling extra features (eg Table Tools) or repetitive init code of various third party plugins.
core/*

All vital JS files can be found here. They are added in all pages and these files, along with the previous app.js file are put in oneui.min.js (found in Get started folder and the dist/ after running the build Grunt task)

In AngularJS version this folder includes a few more scripts which are required for building our web application. When we run Grunts build task, they are all included in oneui.min.js along with the previous app.js, directives.js and controllers.js files.
plugins/*

All supported and integrated third-party plugins are included in here. Some of them require from you to add their stylesheet as well, in the page you would like to use them (always before oneui.css for the style overwrites to work correctly).
pages/*

Custom JS code used on specific pages demonstrating the usage of various plugins and functionality.

<script>

    });
</script>

<div data-toggle="slimscroll" data-height="_scroll area height in pixels eg: 150px_">

</div>

Quick Tips

  • Retina Support

    The UI will look crispy clear on retina screens and if you would like your images to look great, too, then you will have to use the srcset attribute of the <img> element. For example, if you have a regular image img/my_image.jpg and its retina version img/my_image@2x.jpg you will have to use the following markup:

    <img src="img/my_image.jpg" srcset="img/my_image@2x.jpg 2x" alt="Image Description">

    The browser will auto select the best version and load it accordingly. Older browsers will ignore the srcset attribute and will load the regular image as usual.

  • Border-Box

    The Bootstrap framework makes use of the border-box CSS model. That means that if you want to alter the width of an element with CSS, you can do it without worrying about the padding or the border size of that element (they will be adjusted to the width you will set).

  • Focus Styles

    Consider removing .no-focus class form the <html> tag. This way you can restore the default browser focus outline style. Even though custom focus styles have been specified in the template, it is a good practice to keep the focus outline, too, for accessibility purposes.

  • HTTPS

    If you are using https, remember to update the link to Google Web Fonts in the <head> tag accordingly and replace http:// with https://. This change is required for the web fonts to be loaded successfully.

  • Hide Elements from Printing

    Did you know that you can use the class .hidden-print to hide any element you like from printing?

  • User Avatars

    All avatars used in the online demo are not available for commercial use, so I couldn't include them in the final package (they can only be used for demonstration). If you would like them for mockup purposes, you can either replace the images in assets/img/avatars/ folder with avatars from uifaces.com or you can contact me to send you the ones I've used in the online demo (also from uifaces.com). Have in mind that they can only be used for demo purposes (mockup or online).

  • User Names

    All user names used in the template are fictional and only used for demonstration. They were randomly generated at uinames.com, so if you are in need of random names for your mockups, it is a great choice.

  • Favicons

    If you would like to auto create your favicons from your image/logo, you can use the free service at http://realfavicongenerator.net
